# 券商 App 新客开户 A/B 实验复盘

## 项目定位

这个案例围绕一个明确的业务判断展开：

> **这轮流程优化是否足以支持继续买量，并且不会以质量或服务成本为代价。**

项目不是把显著性结果堆在一页上，而是把实验设置、证据检查、机制拆解、分渠道 rollout 规则与灰度验证重点组织成一套能支撑上线决策的分析材料。

---

## 当前结论

当前证据支持继续推进，但**不建议直接全量**。

- 开户完成率：16.82% → 21.55%，绝对提升 **+4.74 ppt**
- 7 日留存率：35.23% → 37.09%，绝对提升 **+1.86 ppt**
- 7 日投诉率：未见显著恶化
- 7 日首充率：34.18% → 36.12%，绝对提升 **+1.94 ppt**

建议先在 uplift 更高、业务弹性更强的渠道灰度推进，再在结果稳定后决定最终上线范围。

## 页面结构

### 首页包含 5 个部分

1. 当前判断与核心业务问题
2. A/B 实验设置与可信度检查
3. 结果总览与机制拆解
4. 分渠道 rollout 方案
5. 全量前仍需验证的风险

### 报告页包含 8 个部分

1. 执行摘要
2. A/B 实验设置与判读规则
3. 指标树与统计口径
4. 可信度检查
5. 整体结果与业务解释
6. 漏斗机制拆解与动作建议
7. 分渠道 rollout 规则
8. 风险、灰度检查单与附录

## 为什么这次替换包比旧版更完整

这版替换包主要解决了旧版的四类问题：

1. **首页与报告页重复度过高**：现在首页负责“判断”，报告页负责“证据链”，角色分工更清楚。
2. **报告不像独立正式文档**：现在报告补上了口径规则、统计方法、可信度检查和附录说明。
3. **README 信息塌成一行**：现在 README 已重写为可阅读、可交付、可解释的仓库说明。
4. **部分表格在源码层挤在一起**：现在主页和报告页都改为结构化 HTML 表格，线上展示更稳定。
