# 券商 App 新客开户实验复盘｜灰度上线决策

本仓库围绕一个核心问题展开：当开户链路的表单负担、流程提示和关键疑问解释被同时优化后，这组改动是否值得继续放量，以及应如何控制上线风险。

## 一、业务问题

券商 App 新客开户通常包含基础资料填写、身份校验、风险测评、绑卡与开户注册等多个高摩擦步骤。对于获客成本较高的业务场景，前中段流失不仅会压缩开户完成数，也会直接影响后续留存、首充与可运营用户池规模。

当前需要回答三个问题：

1. 这组流程优化是否值得继续放量；
2. 效果主要发生在哪个环节；
3. 当前阶段应直接全量，还是先按渠道灰度推进。

## 二、结论摘要

**建议进入分渠道灰度，暂不建议直接全量。**

当前证据支持继续推进，原因主要有三点：

- 开户完成率由 **16.82%** 提升至 **21.55%**，绝对提升 **+4.74 ppt**，95% 置信区间为 **[4.02, 5.45]**；
- 7 日留存率同步提升 **+1.86 ppt**；
- 7 日投诉率未见显著恶化，7 日首充率方向一致。

当前不直接全量，主要因为：

- 本轮改动属于**组合式改动**，只能确认组合效果，不能把 uplift 直接归因到单一设计元素；
- SRM、污染、跨设备识别、成熟窗截断与新鲜感效应仍需在灰度阶段继续验证；
- 7 日留存与 7 日首充属于短期质量信号，不替代首月入金、30 日留存、交易活跃或 LTV。

## 三、关键结果

| 指标 | 对照组 | 实验组 | 绝对变化 | 95% 置信区间 | 业务判断 |
|---|---:|---:|---:|---|---|
| 开户完成率 | 16.82% | 21.55% | +4.74 ppt | [4.02, 5.45] | 支持继续灰度 |
| 7 日留存率 | 35.23% | 37.09% | +1.86 ppt | [0.98, 2.73] | 短期质量同步改善 |
| 7 日投诉率 | 1.277% | 1.280% | +0.00 ppt | [-0.20, 0.21] | 未见显著恶化 |
| 7 日首充率 | 34.18% | 36.12% | +1.94 ppt | [1.07, 2.81] | 商业结果方向一致 |

## 四、漏斗诊断

整体 uplift 的核心，不是最后一步收口改善，而是前中段摩擦下降后，更多用户被推进到链路末端。

| 相邻步骤 | 对照组 | 实验组 | 变化 |
|---|---:|---:|---:|
| Landing → Start onboarding | 80.50% | 80.79% | +0.29 ppt |
| Start → Submit basic info | 69.90% | 77.04% | +7.14 ppt |
| Submit basic info → ID pass | 81.60% | 83.73% | +2.13 ppt |
| ID pass → Risk assessment start | 74.41% | 78.81% | +4.40 ppt |
| Risk start → Risk complete | 72.90% | 78.01% | +5.11 ppt |
| Risk complete → Bind bank card | 81.72% | 82.35% | +0.64 ppt |
| Bind bank card → Complete | 82.63% | 81.69% | -0.94 ppt |

更稳妥的解释是：表单负担下降、流程位置更清晰、关键疑问解释更充分后，基础信息填写和风险测评两段的中断率下降，整体链路推进更顺畅。

末端 `Bind bank card → Complete` 略降，不足以推翻整体 uplift。更合理的解释是，前中段改善扩大了末端分母，而 OTP、绑卡、KYC 复核和外部跳转等待等末端摩擦并未被本轮改动直接解决。

## 五、分渠道灰度策略

| 渠道 | 对照组 | 实验组 | 提升幅度 | 建议动作 |
|---|---:|---:|---:|---|
| 推荐渠道 | 19.95% | 25.60% | +5.65 ppt | 第一优先级 |
| 付费投放 | 12.05% | 17.64% | +5.59 ppt | 第一优先级 |
| 线下客户经理 | 21.79% | 26.77% | +4.98 ppt | 第二梯队 |
| 内容 / 搜索 | 17.48% | 21.90% | +4.41 ppt | 第二梯队 |
| 应用商店 | 15.73% | 19.16% | +3.43 ppt | 保守推进 |

建议按以下顺序推进：

1. 先在推荐渠道与付费投放中继续灰度；
2. 若短期质量指标与渠道 uplift 稳定，再扩展到线下客户经理与内容 / 搜索；
3. 应用商店保守推进，重点补看安装后启动质量与末端收口率。

## 六、建议的灰度通过条件

建议把灰度阶段视为验证结果稳健性的第二道门，而不是机械放量流程。

- 至少覆盖 **14 个自然日**，并在 **7 日成熟窗完整** 后读取留存与首充；
- 开户完成率 uplift 的 **95% 区间下界仍大于 0**；
- 7 日留存率不转负，7 日投诉率未出现持续性恶化；
- assignment、exposure 与分析 cohort 口径一致，不出现明显 SRM；
- 身份验证失败率、客服咨询量、风险问答中断率未显著变差。

## 七、业务价值测算口径

若当前 uplift 在后续灰度中保持稳定，则每 10 万落地页访问用户大致可新增：

- **4,736** 个开户完成；
- **1,858** 个 7 日留存用户；
- **1,940** 个 7 日首充账户。

这些结果用于表达当前 uplift 的业务含义，不直接等于长期收入或生命周期价值预测；若渠道 mix、流量质量或风控门槛变化，应重新估算。
