# 券商 App 新客开户实验复盘报告

## 一、执行摘要

本次实验评估的目标，是验证“表单精简 + 进度条提示 + 关键疑问解释”这一组流程优化，是否能够在不伤害体验护栏的前提下，显著提升券商 App 新客开户完成率。

实验结果显示：

- 开户完成率由 **16.82%** 提升至 **21.55%**，绝对提升 **+4.74 ppt**，95% 置信区间 **[4.02, 5.45]**；
- 7 日留存率提升 **+1.86 ppt**，95% 置信区间 **[0.98, 2.73]**；
- 7 日投诉率变化 **+0.00 ppt**，95% 置信区间 **[-0.20, 0.21]**，未见显著恶化；
- 7 日首充率提升 **+1.94 ppt**，95% 置信区间 **[1.07, 2.81]**。

基于当前证据，建议进入**分渠道灰度上线**，暂不建议直接全量。

## 二、业务背景

券商新客开户是典型的高摩擦业务流程。用户需要完成多步资料填写、身份验证、风险测评与绑卡操作。对于获客成本较高的业务场景，前中段链路是否顺利，直接影响后续留存、首充和可转化用户池规模。

当前需要回答三个核心决策问题：

1. 这组流程优化是否真的提升了开户完成率；
2. 效果主要来自哪个步骤；
3. 当前阶段应直接全量，还是先进入灰度上线。

- 主指标改善；
- 护栏未见明显恶化；
- 留存与首充方向一致。

因此，这轮实验更像是在**降低流程摩擦**，而不是通过误导、催促或压缩信息质量换取短期转化。

## 六、漏斗诊断

### 6.1 累计阶段到达率

实验组在多个阶段的累计到达率均高于对照组，说明整体链路推进更顺畅。

### 6.2 相邻步骤转化率

| 相邻步骤 | 对照组 | 实验组 | 绝对变化 |
|---|---:|---:|---:|
| Landing → Start onboarding | 80.50% | 80.79% | +0.29 ppt |
| Start → Submit basic info | 69.90% | 77.04% | **+7.14 ppt** |
| Submit basic info → ID pass | 81.60% | 83.73% | +2.13 ppt |
| ID pass → Risk assessment start | 74.41% | 78.81% | **+4.40 ppt** |
| Risk start → Risk complete | 72.90% | 78.01% | **+5.11 ppt** |
| Risk complete → Bind bank card | 81.72% | 82.35% | +0.64 ppt |
| Bind bank card → Complete | 82.63% | 81.69% | **-0.94 ppt** |

### 6.3 核心诊断

这组结果说明：

1. **基础信息填写**是最明显的改善点，表单精简直接降低了放弃率；
2. **风险测评进入与完成**也明显提升，说明进度条提示与关键疑问解释有效降低了理解成本；
3. **最后一步并不是核心贡献来源**，甚至略有下降，说明整体提升主要来自前中段流程推进，而非末端收口优化。

### 6.4 为什么最后一步略降，整体结论仍然成立

最后一步 `Bind bank card → Complete` 的相邻步骤转化率下降 **0.94 ppt**，但这并不与整体结论冲突，原因主要有三点：

- **分母构成变化**：当前改动把更多中等意向用户推进到了末端，因此最后一步的分母扩大，末端收口率被轻微摊薄是可能的；
- **末端摩擦未被直接解决**：OTP、银行卡验证、KYC 复核、外部跳转等待等因素仍然存在，并不是当前改动的主要作用点；
- **量级不对称**：末端 -0.94 ppt 明显小于前中段 +7.14 / +4.40 / +5.11 ppt 的改善幅度，因此不会推翻端到端完成率的显著提升。

更合理的业务解释是：这轮改动先把更多用户送到了末端，但末端收口摩擦仍然独立存在。

## 八、灰度上线建议

### 8.1 第一阶段

优先在 **推荐渠道 + 付费投放** 渠道灰度上线。

### 8.2 第二阶段

若护栏保持稳定，再扩展到线下客户经理与内容 / 搜索。

### 8.3 灰度期间持续跟踪的指标

- 7 日投诉率
- 身份验证失败率
- 客服咨询量
- 7 日首充率
- 高价值用户激活质量

### 8.4 为什么不建议直接全量

因为当前这组改动由多项改动打包构成，且开户注册链路受合规、身份校验与服务资源约束较强，直接全量上线会降低归因清晰度，也会扩大潜在风险暴露。

## 九、实验风险与有效性威胁

- **SRM（样本比例失衡）**：当前样本比例未见明显异常，但正式环境中仍应作为第一道健康检查。
- **污染 / 干扰**：用户跨渠道回流、客服人工补救、重复进入链路，都会稀释组间边界。
- **跨设备识别问题**：用户在不同设备间切换时，若身份未正确合并，可能出现跨组暴露或步骤漏记。
- **观察窗截断**：7 日留存和 7 日首充需要完整成熟窗，尾部样本若未成熟会系统性低估结果。
- **新鲜感效应**：进度条与 FAQ 类改动，可能在实验早期产生短期抬升，后续需按周观察效果漂移。

这些风险不会自动推翻当前结果，但会影响“是否适合直接全量”的判断，因此更适合先灰度、再验证稳定性。
