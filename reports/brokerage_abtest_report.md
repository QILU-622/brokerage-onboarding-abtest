# 分析报告｜券商 App 新客开户 A/B 实验复盘

## 一、执行摘要

当前证据支持继续推进，但不支持直接全量。

开户完成率提升 +4.74 ppt，7 日留存与 7 日首充方向一致，投诉率未见显著恶化；但长期价值尚未被验证，且过程层护栏、SRM、污染与跨设备识别仍需在灰度阶段继续确认。

## 二、业务问题定义

1. 当前 uplift 是否可信，是否足以支持继续推进。
2. 效果主要来自哪些环节，哪些摩擦被缓解，哪些仍未解决。
3. 哪些渠道更适合先放量，放量规则与回滚规则如何写清楚。
4. 哪些实验风险会直接影响是否适合直接全量。

## 三、A/B 实验设置与判读规则

- 对照组：原开户流程
- 实验组：本轮优化后的组合版本
- 随机分流：用户层级 50/50 分流
- 主指标：开户完成率
- 护栏指标：7 日留存率、7 日投诉率
- 次级指标：7 日首充率
- 判读规则：统一使用绝对百分点变化（ppt），整体结果使用双样本比例检验与非合并 95% 置信区间；分渠道 uplift 使用 Holm 校正

## 四、可信度检查

- Balance check：channel p=0.205 · device p=0.286 · intent p=0.421 · age p=0.936
- Mature window：7 日留存与 7 日首充只能在 mature cohort 上读取
- assignment → exposure → outcome：灰度阶段补齐首曝日志、曝光日志与用户去重规则

## 五、整体结果

| 指标 | 对照组 | 实验组 | 绝对变化 | 95% 置信区间 | 业务判断 |
|---|---:|---:|---:|---|---|
| 开户完成率 | 16.82% | 21.55% | +4.74 ppt | [4.02, 5.45] | 支持继续灰度 |
| 7 日留存率 | 35.23% | 37.09% | +1.86 ppt | [0.98, 2.73] | 短期质量同步改善 |
| 7 日投诉率 | 1.277% | 1.280% | +0.00 ppt | [-0.20, 0.21] | 未见显著恶化 |
| 7 日首充率 | 34.18% | 36.12% | +1.94 ppt | [1.07, 2.81] | 商业方向一致 |

## 六、机制拆解

| 步骤 | 对照组 | 实验组 | 绝对变化 |
|---|---:|---:|---:|
| Landing → Start onboarding | 80.50% | 80.79% | +0.29 ppt |
| Start → Submit basic info | 69.90% | 77.04% | +7.14 ppt |
| Submit basic info → ID pass | 81.60% | 83.73% | +2.13 ppt |
| ID pass → Risk assessment start | 74.41% | 78.81% | +4.40 ppt |
| Risk start → Risk complete | 72.90% | 78.01% | +5.11 ppt |
| Risk complete → Bind bank card | 81.72% | 82.35% | +0.64 ppt |
| Bind bank card → Complete | 82.63% | 81.69% | -0.94 ppt |

当前更稳妥的解释是：本轮改动降低了前中段认知负担和流程摩擦，推动更多用户走到链路末端。

## 七、分渠道 rollout 方案

- 第一优先级：推荐渠道、付费投放
- 第二梯队：线下客户经理、内容 / 搜索
- 保守推进：应用商店

放量前必须把升级与回滚规则写清楚，并在灰度阶段固定检查过程层护栏。

## 八、风险与下一步

- SRM
- 污染 / 干扰
- 跨设备识别问题
- 成熟窗截断
- 新鲜感效应

下一轮应先按渠道灰度推进，再拆解单因素测试，并补齐首曝日志、曝光日志、人工介入标签和过程层护栏。
