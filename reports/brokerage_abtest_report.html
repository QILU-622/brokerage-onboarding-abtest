一、业务问题定义

问题 01

这轮流程优化是否足以支持继续买量，并保持质量与服务成本稳定？

问题 02

uplift 主要来自链路的哪个阶段，哪些摩擦被缓解，哪些仍未解决？

问题 03

在不同渠道下，是否已经具备可复制的放量条件，以及应该如何设置升级与回滚规则？

券商开户链路包含资料填写、身份校验、风险测评、绑卡与开户注册等多个高摩擦环节。 在获客成本较高的场景里，本轮改动只有在同时提升 全链路完成率、且不引入更差的质量或服务成本时，才具备继续扩量价值。

二、A/B 实验设置与判读规则

实验最基础的四个定义

概念当前设置为什么重要
对照组（Control）原开户流程提供同一时期下的比较基线，避免把季节性、投放波动或用户结构变化误当成改版效果
实验组（Treatment）本轮优化后的组合版本当前结论对应整套新流程的净效果，暂不拆到单个组件
随机分流用户层级 50/50 分流尽可能让两组在基础结构上可比，支撑 uplift 的因果解释
结果判读主指标 + 护栏指标 + 次级指标 + 风险边界上线节奏同时参考收益、质量和执行条件，避免只看转化率

  • 护栏指标：7 日留存率与 7 日投诉率。
  • 次级指标：7 日首充率。

发现推断下一步动作
基础信息填写阶段 uplift 最大表单负担下降后，开始填写的用户更容易顺利完成资料提交拆分验证字段裁剪、默认值填充、证件识别辅助的独立贡献
风险测评启动与完成均明显改善流程位置更清晰、解释更充分后，用户更容易完成问答分别测试 FAQ 呈现位置、进度提示方式和说明文案粒度
末端收口轻微回落前中段改善扩大了末端分母，但 OTP / 绑卡 / KYC 摩擦并未被直接解决独立排查 OTP 失败率、绑卡失败率、KYC 复核和外部跳转等待时长
为什么最后一步略降，整体结论仍然成立
  • 分母构成变化：前中段改善把更多中等意向用户送到末端，收口率被轻微摊薄是可能的。
  • 末端摩擦未被直接解决：OTP、银行卡验证、KYC 复核与外部跳转等待等因素仍然存在。
  • 量级不对称：末端回落幅度明显小于前中段几个关键节点的提升幅度，因此不改变整体方向。

七、实验风险与下一步

需要继续验证的风险

  • SRM：样本比例失衡或 assignment / exposure / cohort 口径不一致，会削弱 uplift 解释力。
  • 污染 / 干扰：用户跨渠道回流、重复进入链路或客服人工补救，都会稀释组间边界。
  • 跨设备识别问题：用户多设备切换时，若身份未正确合并，可能产生跨组暴露或步骤漏记。
  • 成熟窗截断：7 日留存和 7 日首充必须在完整 成熟 cohort 上读取。
  • 新鲜感效应：FAQ 与进度提示类改动可能在实验早期高估 uplift，需要按周观察 effect drift。

下一轮建议

  1. 先按渠道灰度推进，验证 uplift 稳定性和质量约束，再决定是否扩大范围。
  2. 拆解单因素测试，分别验证表单精简、进度提示与 FAQ 解释的独立贡献。
  3. 补充首曝日志、曝光日志、人工介入标签，形成完整 assignment → exposure → outcome 链条。
  4. 新增并接入过程层护栏：身份验证失败率、OTP 失败率、绑卡失败率、风险问答中断率、客服咨询量。
  5. 补长期结果：首月入金、首月交易活跃、30 日留存，并按周跟踪 effect drift。
注：报告数值来自脱敏重建场景，不对应任何实际机构经营数据。所有百分比变化均为绝对百分点变化（ppt）。
