这个项目不是把 A/B 实验做成一页“显著性展示”，而是把结果真正转成业务动作： 是否值得放量、效果主要来自哪里、为什么当前建议先灰度而不是直接全量。

决策摘要

推荐动作

建议进入分渠道灰度，不建议直接全量。

当前证据为什么支持灰度

主指标开户完成率从 16.82% 提升到 21.55%，绝对 uplift 为 +4.74 ppt，95% CI 为 [4.02, 5.45]。同时，7 日留存提升 +1.86 ppt，投诉率未见显著恶化，说明效果并非建立在明显体验受损之上。

为什么不是直接全量

当前 treatment 属于表单精简 + 进度条提示 + FAQ 解释的组合式改动，只能确认组合效果，不能直接识别单项设计元素的独立贡献；此外，SRM、污染、跨设备识别、成熟窗截断与 novelty effect 仍需在灰度阶段继续验证。

业务含义

这轮实验更像是一次降低前中段认知负担的结构性改动，而不是单纯优化最后确认页的收口动作。

最重要的机制发现

uplift 的主来源不是“最后一步更好关单”，而是基础信息填写与风险测评阶段的推进效率更高。

最大改善发生在 Start → Submit basic info (+7.14 ppt)、ID pass → Risk assessment start (+4.40 ppt) 与 Risk start → Risk complete (+5.11 ppt)。
因此当前更合理的产品动作不是“直接全量上线然后结束”，而是：先灰度验证稳定性，再拆解单因素实验做归因。

潜在业务价值

若 uplift 在后续灰度中保持稳定，则每 10 万 landing 用户约可多带来 4,736 个开户完成、1,858 个 7 日留存 与 1,940 个 7 日首充。

业务问题

券商 App 新客开户链路通常包含基础资料填写、身份校验、风险测评、绑卡与开户注册等多个高摩擦步骤。对于 acquisition 成本较高的业务场景，用户是否能顺利穿过前中段链路，直接影响后续留存、入金与可转化用户池规模。

本案例要回答的不是“实验有没有显著”，而是三个更接近业务现场的问题：

  • 这组流程优化是否值得继续放量？
  • uplift 究竟发生在链路的哪个环节？
  • 下一轮应该先拆哪一个因素，而不是继续笼统堆改动？

核心假设是：如果通过表单精简、进度提示与关键疑问解释降低认知成本，那么开户完成率应显著提升，且不应以明显伤害体验护栏为代价。

实验设计

实验框架

Randomization：user-level 50/50 分流。Control = 23,256，Treatment = 22,962。

Window：2026-01-01 至 2026-03-16，共 75 天，覆盖完整开户链路与 7 日后验指标。

指标体系

  • Primary：开户完成率
  • Guardrails：7 日留存率、7 日投诉率
  • Exploratory：7 日首充率

统计方法

Balance check + 双样本比例检验 + 95% CI；渠道分层结果额外做 Holm 多重比较校正。

平衡性检查

渠道、设备、来源意图、年龄段分布均未见显著失衡，说明观察到的 uplift 更可能来自流程改动本身，而不是样本结构偏差。

  • channel: p = 0.205
  • device: p = 0.286
  • intent: p = 0.421
  • age: p = 0.936

样本识别能力

按当前样本规模估算，在 α = .05、80% power 条件下，主指标约可识别 0.99 ppt 的绝对 uplift；本次实际观察到的 uplift 为 4.74 ppt。

这意味着本次结果不是“刚好显著”，而是 observed effect 明显高于当前样本规模下的可识别下限。

实验结果

核心指标同时展示 effect size、95% CI 与业务解释。重点不是只说“显著”，而是说明结论如何支持后续动作。

机制拆解：uplift 主要来自哪里

为了定位效果来源，需要同时观察累计 reach 与相邻步骤转化率，避免把“更多人走到后面”与“某一步本身更顺”混为一谈。

核心诊断

  • 最大贡献点：Start → Submit basic info 提升 +7.14 ppt，说明表单精简显著降低了填写摩擦与放弃率。
  • 第二贡献点：ID pass → Risk assessment start 与 Risk start → Risk complete 分别提升 +4.40 ppt 和 +5.11 ppt，说明进度提示与 FAQ 解释降低了理解和决策负担。
  • 结论：本轮 uplift 主要来自前中段摩擦下降，而不是最后一步收口改善。

为什么最后一步略降，整体结论仍然成立

现象：bind bank card → account open complete 的 step conversion 略降 0.94 ppt，看起来与整体正向 uplift 相反。

  • 分母构成变化：treatment 把更多中等意向用户推进到了末端，最后一步分母扩大，末端收口率被轻微摊薄是可能的。
  • 末端摩擦未被直接解决：OTP、银行卡验证、KYC 复核、外部跳转等待等因素仍然存在，这些并不是当前 treatment 的主要作用点。
  • 量级不对称：末端 -0.94 ppt 明显小于前中段 +7.14 / +4.40 / +5.11 ppt 的改善，因此不会推翻 end-to-end completion 的显著提升。
更成熟的解释不是“最后一步也优化了”，而是：当前 treatment 先把更多用户成功送到末端，但末端收口摩擦仍然独立存在。

灰度策略：先在哪些渠道放量

更成熟的做法不是默认全渠道一起推，而是先看分层效果，再决定灰度顺序与资源投入重点。

灰度逻辑

  • Referral / Paid social 优先：绝对 uplift 更高，说明对高摩擦来源用户更有效，资源投入回报更直接。
  • 不直接全量：当前 treatment 仍受组合改动、合规链路、KYC 成功率与客服承压影响，先灰度更接近真实业务决策。
  • 灰度阶段继续跟踪：7 日投诉率、身份验证失败率、客服咨询量、首充率、成熟 cohort 质量与 effect drift。

局限性与下一步

当前局限性

  • 本案例使用 synthetic data 复现实验分析流程，相关结论仅用于方法演示与作品集展示。
  • 当前 treatment 为多项改动打包，因此只能识别组合效果，不能直接识别每个设计元素的独立贡献。
  • 7 日留存与 7 日首充仍属于短期质量指标，不能替代长期入金质量、交易活跃或客户生命周期价值。
  • SRM、污染、跨设备、成熟窗截断与 novelty effect 在当前案例中被显性化讨论，但并不意味着已经被现有数据完全排除。

下一步实验建议

  • 拆解单因素测试：表单精简 / 进度条 / FAQ 解释。
  • 新增护栏：身份验证失败率、客服咨询量、风险问答中断率。
  • 新增日志：首曝日志、曝光日志、客服人工介入标签、cross-device stitching 质量标记。
  • 新增分层：新老客、设备类型、来源意图、是否高价值潜客。
  • 补长期指标：首月入金、首月交易活跃、30 日留存，并按周跟踪 effect drift。
