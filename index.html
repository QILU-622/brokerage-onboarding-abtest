三、漏斗归因

整体 uplift 主要来自前中段摩擦下降，而不是末端收口改善。下一轮动作应优先围绕资料填写与风险测评环节继续拆解。

发现 → 推断 → 动作

发现推断下一步动作
Start → Submit basic info uplift 最大（+7.14 ppt） 表单负担下降后，进入填写的用户更容易完成资料提交 拆分验证字段裁剪、默认值填充、证件识别辅助的独立贡献
Risk assessment start / complete 同步提升 流程位置更清晰、解释更充分后，用户更容易进入并完成风险问答 分别测试 FAQ 呈现位置、进度提示方式和说明文案粒度
Bind bank card → Complete 轻微回落（-0.94 ppt） 前中段改善扩大了末端分母，但 OTP、绑卡与 KYC 摩擦未被本轮改动直接覆盖 单独排查 OTP 失败率、绑卡失败率、KYC 复核与外部跳转等待时长
为什么最后一步略降，整体结论仍成立？
  • 前中段改善把更多中等意向用户送到末端，末端分母扩大后，收口率被轻微摊薄是可能的。
  • OTP、银行卡验证、KYC 复核与外部跳转等待等末端摩擦不属于本轮改动的主要作用点。
  • 末端回落量级明显小于前中段几个关键节点的提升量级，因此不改变 全链路完成率 的整体方向。

四、分渠道灰度推进方案

五、实验有效性与剩余风险

这些风险直接影响是否适合直接全量，因此必须在灰度阶段固定检查。

可信度检查

检查项当前观察为什么重要
平衡性检查channel p=0.205 · device p=0.286 · intent p=0.421 · age p=0.936保证实验组与对照组基础结构可比，降低样本结构差异污染
SRM / cohort 规则当前未见明显异常，但需继续监控 assignment → exposure → outcome 链条一旦样本比例失衡或口径错位，整体 uplift 解释力会明显下降
成熟窗7 日留存与 7 日首充只能在 成熟 cohort 上读取避免尾部样本未成熟导致后验指标被系统性低估
Contamination / cross-device灰度阶段继续检查跨渠道回流、重复进入链路、跨设备识别与人工补救标签防止组间边界被稀释，结果可解释性下降

灰度阶段必须补齐

  • 首曝日志、曝光日志、用户去重规则，形成 assignment → exposure → outcome 闭环。
  • 身份验证失败率、OTP 失败率、绑卡失败率、风险问答中断率。
  • 客服咨询量、人工补救标签与投诉分类规则，用于识别是否出现服务成本上升。
  • 30 日留存、首月入金、首月交易活跃与渠道分层质量结果。

下一轮实验优先级

  1. 先在重点渠道验证当前组合方案稳定性，再决定是否扩大范围。
  2. 把表单精简、进度提示、FAQ 解释拆成单因素实验，收紧归因结论。
  3. 独立排查末端摩擦，避免前中段改善被 OTP / KYC / 绑卡失败吞噬。
  4. 把结果层、质量层、过程层护栏接入同一张运营看板。

六、经营含义

当前 uplift 不等于长期收入预测，但足以帮助判断同等流量规模下，是否更可能改善预算效率与有效账户产出。
